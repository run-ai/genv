Note

This page talks about installing Genv on a single machine. If you are interested in using Genv on a multi-machine cluster, check out the remote installation guide.

Core¶

The core components of Genv are part of the Genv Python package. Install it with the following command:

pip install genv

Warning

If you see a warning message similar to the following, add the specified directory to your $PATH by editing your ~/.bashrc and restarting your shell:

WARNING: The script genvctl is installed in '/home/raz/.local/bin' which is not on PATH.
Consider adding this directory to PATH...

Note

Genv needs Python 3.7 or higher so make sure you have it installed.

The Python package should also install the Genv CLI. Verify the installation with the command:

genvctl --help

Warning

If you see genvctl: command not found, your $PATH is probably no set as explained above.

Terminal¶

To use Genv in your shell, you will need to install the terminal components of Genv.

Note

Before installing the terminal components, make sure you have the core components of Genv installed.

You can install the terminal components of Genv in several ways.

Conda¶

If you are using Conda, you can install the genv package from the channel conda-forge:

conda install -c conda-forge genv

From Source¶

If you are not using Conda, you can clone the Genv repository to your home directory:

git clone https://github.com/run-ai/genv.git $HOME/genv

Then, you will need to set up your shell by adding the following commands to your ~/.bashrc or any other equivalent file:

export PATH=$HOME/genv/bin:$PATH
eval "$(genv init -)"

Afterward, for this to take effect, either reopen your terminal or restart your shell using the command:

exec $SHELL

To verify the installation, run the following command:

genv

You should be able to see all the available Genv commands.

Uninstalling¶

To uninstall the terminal components of Genv, remove the commands you added to your ~/.bashrc or any other equivalent file.

Then, remove its root directory:

rm -rf $(genv root)

PyCharm¶

Currently, there is no PyCharm plugin for Genv. This is however part of the project roadmap.

In case you use PyCharm, please open an issue in the project repository.

This will help us prioritize this as well as suggest other ways to work with Genv in PyCharm in the meantime.
